## Описание системных вызовов Linux

### `write`
Записывает данные из буфера в файловый дескриптор (файл, сокет, пайп и т.д.). Возвращает количество записанных байт.

### `mmap`
Отображает файл или устройство в память процесса. Используется для выделения памяти, загрузки библиотек и межпроцессного взаимодействия.

### `ioctl`
Универсальный интерфейс управления устройствами. Позволяет отправлять специфичные команды драйверам устройств, которые не вписываются в стандартные вызовы `read`/`write`.

### `openat`
Открывает файл относительно директории, указанной файловым дескриптором. Более безопасная версия `open`, устойчивая к race condition при работе с путями.

### `getpgrp`
Возвращает идентификатор группы процессов (PGID) текущего процесса. Группы процессов используются для управления терминалом и сигналами.

### `read`
Читает данные из файлового дескриптора в буфер. Возвращает количество прочитанных байт или 0 при достижении конца файла.

### `execve`
Заменяет текущий процесс новой программой. Загружает исполняемый файл, передаёт аргументы и переменные окружения. Старый код процесса полностью заменяется новым.

### `connect`
Устанавливает соединение с удалённым адресом через сокет (TCP-соединение или подключение к UNIX-сокету).

### `close`
Закрывает файловый дескриптор, освобождая связанные с ним ресурсы ядра. После закрытия дескриптор становится недействительным.

### `pselect6`
Ожидает готовности одного или нескольких файловых дескрипторов к чтению, записи или наступления ошибки. Поддерживает атомарное изменение маски сигналов на время ожидания.

### `mprotect`
Изменяет права доступа к области памяти (чтение, запись, исполнение). Используется для защиты страниц памяти.

### `fstat`
Возвращает информацию о файле по его файловому дескриптору: размер, права доступа, время изменения, тип файла и т.д.

### `newfstatat`
Расширенная версия `fstat`. Возвращает информацию о файле по пути относительно указанного директорного дескриптора.

### `socket`
Создаёт новый сокет заданного типа (TCP, UDP, UNIX и др.) и возвращает его файловый дескриптор.

### `recvfrom`
Получает данные из сокета. В отличие от `recv`, также сохраняет адрес отправителя. Используется преимущественно с UDP.

### `clock_gettime`
Возвращает текущее время указанных часов (реального времени, монотонных, процессорного времени и др.) с наносекундной точностью.

### `sendmmsg`
Отправляет сразу несколько сообщений через сокет за один системный вызов. Более эффективная версия `sendmsg` для пакетной отправки.

### `poll`
Ожидает события на наборе файловых дескрипторов (готовность к чтению/записи, ошибки). Аналог `select`, но без ограничения на количество дескрипторов.

### `rt_sigprocmask`
Изменяет маску заблокированных сигналов текущего потока. Позволяет блокировать, разблокировать или заменять набор игнорируемых сигналов.

### `getsockname`
Возвращает локальный адрес (IP и порт), к которому привязан сокет. Полезно для определения порта после автоматического назначения системой.

### `munmap`
Освобождает область памяти, ранее отображённую через `mmap`. Противоположность `mmap`.

### `sendto`
Отправляет данные через сокет на указанный адрес. Используется преимущественно с UDP, где адрес указывается при каждой отправке.

### `recvmsg`
Получает сообщение из сокета с расширенными возможностями: поддерживает scatter/gather I/O и передачу вспомогательных данных (ancillary data).

### `brk`
Изменяет размер сегмента данных процесса (heap). Устанавливает новый конец кучи. Используется реализацией `malloc` в libc.

### `getuid`
Возвращает реальный идентификатор пользователя (UID) текущего процесса. Отличается от эффективного UID, используемого при проверке прав.

### `lseek`
Изменяет позицию чтения/записи в файле. Позволяет перемещаться по файлу на абсолютную или относительную позицию.

### `futex`
Реализует быстрые мьютексы в пространстве пользователя (Fast Userspace Mutex). Основа для реализации `pthread_mutex`, семафоров и других примитивов синхронизации.

### `rt_sigaction`
Устанавливает обработчик сигнала для текущего процесса. Позволяет задать функцию-обработчик, маску блокируемых сигналов и флаги поведения.

### `bind`
Привязывает сокет к локальному адресу (IP и порту). Обязательный шаг перед вызовом `listen` для серверного сокета.

### `statfs`
Возвращает информацию о файловой системе по указанному пути: тип ФС, размер блока, количество свободных блоков и инодов.

### `access`
Проверяет доступность файла для текущего процесса по реальному UID/GID. Проверяет существование файла и права на чтение, запись или исполнение.

### `setsockopt`
Устанавливает параметры сокета: таймауты, размеры буферов, режим повторного использования адреса (`SO_REUSEADDR`) и др.

### `uname`
Возвращает информацию о системе: имя ядра, имя хоста, версию ядра, аппаратную платформу.

### `set_tid_address`
Устанавливает адрес, по которому ядро запишет TID (идентификатор потока) при завершении потока. Используется библиотекой pthreads для реализации `pthread_join`.

### `getrandom`
Заполняет буфер случайными байтами из криптографически стойкого генератора ядра. Более надёжная альтернатива чтению из `/dev/urandom`.

### `pread64`
Читает данные из файла по заданному смещению без изменения текущей позиции файла. Атомарно по отношению к другим потокам.

### `getpeername`
Возвращает адрес удалённой стороны подключённого сокета (IP и порт партнёра по TCP-соединению).

### `arch_prctl`
Устанавливает или получает архитектурно-специфичные параметры потока. На x86-64 используется для установки базовых адресов сегментных регистров `FS` и `GS` (например, для TLS — thread-local storage).

### `set_robust_list`
Регистрирует список «надёжных» мьютексов потока. Ядро автоматически освобождает эти мьютексы при аварийном завершении потока, предотвращая взаимоблокировки.

### `rseq`
Регистрирует структуру «restartable sequences» — механизм для реализации атомарных операций в пространстве пользователя без системных вызовов. Ускоряет работу с per-CPU данными.

### `getgroups`
Возвращает список дополнительных групповых идентификаторов (GID) текущего процесса.

### `prlimit64`
Получает или устанавливает ресурсные ограничения процесса (лимиты на память, количество файлов, размер стека и др.). Расширенная версия `setrlimit`/`getrlimit` с поддержкой 64-битных значений.
