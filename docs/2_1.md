# 2.1

## 1. Запуск программы
```
execve("./hello", ["./hello"], 0x7ffe35669eb0 /* 59 vars */) = 0
```
**Назначение**: Запуск исполняемого файла `./hello`
- Первый аргумент: путь к программе
- Второй: массив аргументов командной строки
- Третий: указатель на переменные окружения (59 переменных)
- Возвращает 0 при успехе

## 2. Инициализация кучи
```
brk(NULL) = 0x564324a8a000
```
**Назначение**: Получение текущего конца сегмента данных процесса (heap)
- NULL означает запрос текущего значения без изменений
- Возвращает адрес конца кучи

## 3. Поиск файла предзагрузки
```
access("/etc/ld.so.preload", R_OK) = -1 ENOENT (No such file or directory)
```
**Назначение**: Проверка существования файла с библиотеками для предзагрузки
- R_OK проверяет право на чтение
- Файл отсутствует (ENOENT), это нормально

## 4. Загрузка кэша динамического линковщика
```
openat(AT_FDCWD, "/etc/ld.so.cache", O_RDONLY|O_CLOEXEC) = 3
```
**Назначение**: Открытие кэша с путями к разделяемым библиотекам
- AT_FDCWD: относительно текущей директории
- O_RDONLY: только для чтения
- O_CLOEXEC: закрыть при execve
- Возвращает файловый дескриптор 3

```
fstat(3, {st_mode=S_IFREG|0644, st_size=175299, ...}) = 0
```
**Назначение**: Получение информации о файле (размер, права)
- Размер файла: 175299 байт
- Права: 0644 (rw-r--r--)

```
mmap(NULL, 175299, PROT_READ, MAP_PRIVATE, 3, 0) = 0x7f8cdbd4e000
```
**Назначение**: Отображение кэша в память для быстрого доступа
- PROT_READ: только чтение
- MAP_PRIVATE: изменения не влияют на файл
- Размещен по адресу 0x7f8cdbd4e000

```
close(3) = 0
```
**Назначение**: Закрытие дескриптора кэша (данные уже в памяти)

## 5. Загрузка стандартной библиотеки C
```
openat(AT_FDCWD, "/usr/lib/libc.so.6", O_RDONLY|O_CLOEXEC) = 3
```
**Назначение**: Открытие libc - основной библиотеки C

```
read(3, "\177ELF\2\1\1\3...", 832) = 832
```
**Назначение**: Чтение ELF заголовка библиотеки
- \177ELF - магическое число ELF файла
- Читает первые 832 байта

```
pread64(3, "\6\0\0\0\4\0\0\0@\0\0\0...", 840, 64) = 840
```
**Назначение**: Чтение program headers с позиции 64
- pread64: позиционное чтение (не меняет смещение файла)

```
fstat(3, {st_mode=S_IFREG|0755, st_size=2010168, ...}) = 0
```
**Назначение**: Информация о libc.so.6
- Размер: ~2MB
- Права: 0755 (исполняемый файл)

## 6. Отображение libc в память
```
mmap(NULL, 8192, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS, -1, 0) = 0x7f8cdbd4c000
```
**Назначение**: Выделение анонимной памяти для служебных структур
- 8KB на чтение и запись
- MAP_ANONYMOUS: без файла, просто память

```
pread64(3, "...", 840, 64) = 840
```
**Назначение**: Повторное чтение program headers для анализа

```
mmap(NULL, 2034544, PROT_READ, MAP_PRIVATE|MAP_DENYWRITE, 3, 0) = 0x7f8cdbb5b000
```
**Назначение**: Отображение всей библиотеки в память
- ~2MB только для чтения
- MAP_DENYWRITE: запретить запись в файл

```
mmap(0x7f8cdbb7f000, 1511424, PROT_READ|PROT_EXEC, MAP_PRIVATE|MAP_FIXED|MAP_DENYWRITE, 3, 0x24000) = 0x7f8cdbb7f000
```
**Назначение**: Отображение сегмента кода libc
- PROT_EXEC: разрешение на исполнение
- MAP_FIXED: по конкретному адресу
- Смещение 0x24000 в файле

```
mmap(0x7f8cdbcf0000, 319488, PROT_READ, MAP_PRIVATE|MAP_FIXED|MAP_DENYWRITE, 3, 0x195000) = 0x7f8cdbcf0000
```
**Назначение**: Отображение сегмента read-only данных
- Константы, строки и т.д.

```
mmap(0x7f8cdbd3e000, 24576, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_FIXED|MAP_DENYWRITE, 3, 0x1e2000) = 0x7f8cdbd3e000
```
**Назначение**: Отображение сегмента данных
- Глобальные и статические переменные
- 24KB с правами чтения и записи

```
mmap(0x7f8cdbd44000, 31600, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_FIXED|MAP_ANONYMOUS, -1, 0) = 0x7f8cdbd44000
```
**Назначение**: Дополнительная память для BSS сегмента
- Неинициализированные данные

```
close(3) = 0
```
**Назначение**: Закрытие libc.so.6 (уже отображен в памяти)

## 7. Настройка потока выполнения
```
mmap(NULL, 12288, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS, -1, 0) = 0x7f8cdbb58000
```
**Назначение**: Выделение памяти для структур потока (thread-local storage)
- 12KB для TLS

```
arch_prctl(ARCH_SET_FS, 0x7f8cdbb58740) = 0
```
**Назначение**: Установка регистра FS для thread-local storage
- FS указывает на TLS текущего потока

```
set_tid_address(0x7f8cdbb58d68) = 84495
```
**Назначение**: Установка адреса для Thread ID
- Возвращает TID процесса (84495)
- Используется для уведомления о завершении потока

```
set_robust_list(0x7f8cdbb58a20, 24) = 0
```
**Назначение**: Установка списка robust futex
- Для корректной обработки блокировок при аварийном завершении потока

```
rseq(0x7f8cdbb586a0, 0x20, 0, 0x53053053) = 0
```
**Назначение**: Регистрация restartable sequences
- Оптимизация для lock-free алгоритмов
- Позволяет перезапустить операцию при вытеснении потока

## 8. Защита памяти
```
mprotect(0x7f8cdbd3e000, 16384, PROT_READ) = 0
```
**Назначение**: Изменение прав доступа к данным libc
- Делает часть данных только для чтения (защита от перезаписи)

```
mprotect(0x5642e9af0000, 4096, PROT_READ) = 0
```
**Назначение**: Защита GOT (Global Offset Table) программы
- Предотвращает атаки на таблицу глобальных смещений

```
mprotect(0x7f8cdbdb7000, 8192, PROT_READ) = 0
```
**Назначение**: Защита данных динамического линковщика

## 9. Проверка ограничений

```
prlimit64(0, RLIMIT_STACK, NULL, {rlim_cur=8192*1024, rlim_max=RLIM64_INFINITY}) = 0
```
**Назначение**: Получение лимита на размер стека
- Текущий лимит: 8MB

**Назначение**: Проверка типа файла дескриптора 1 (stdout)
- S_IFCHR: символьное устройство (терминал)

## 13. Расширение кучи

# ASLR и TLS - подробное объяснение

## ASLR (Address Space Layout Randomization)

**ASLR** - это механизм безопасности, который рандомизирует расположение ключевых областей памяти процесса при каждом запуске программы.

### Что рандомизируется:
- **Стек** - область для локальных переменных
- **Куча (heap)** - динамически выделяемая память
- **Разделяемые библиотеки** - libc, libpthread и др.
- **Исполняемый код** (если скомпилирован как PIE)

### Зачем это нужно:
```
Без ASLR:
[Код программы: 0x400000] [Библиотеки: 0x7f000000] [Стек: 0x7fff0000]
                ↑                    ↑                    ↑
         Всегда одинаковые адреса - легко атаковать!

С ASLR:
Запуск 1: [0x56a341000] [0x7f8cdbb5b000] [0x7ffe35669000]
Запуск 2: [0x55f892000] [0x7fb21a3c4000] [0x7ffd9a8b2000]
Запуск 3: [0x564324000] [0x7f8cdbb5b000] [0x7ffe35669000]
                ↑ Адреса меняются при каждом запуске!
```

### В вашем trace.txt:
```
getrandom("\x24\xff\x0f\x17\xbd\xef\x2a\x9c", 8, GRND_NONBLOCK) = 8
```
Эти случайные байты используются для генерации случайных адресов.

### Проверка ASLR:
```bash
# Проверить статус ASLR
cat /proc/sys/kernel/randomize_va_space
# 0 = выключен, 1 = частичный, 2 = полный

# Запустить программу несколько раз и посмотреть адреса
./hello & cat /proc/$!/maps
./hello & cat /proc/$!/maps
# Адреса будут разными!
```

### Защита от атак:
- **Buffer overflow** - сложнее угадать адрес возврата
- **Return-to-libc** - адреса функций libc меняются
- **ROP (Return-Oriented Programming)** - сложнее найти gadgets

## TLS (Thread-Local Storage)

**TLS** - это механизм, позволяющий каждому потоку иметь свою копию глобальных переменных.

### Концепция:
```c
// Обычная глобальная переменная - общая для всех потоков
int global_counter = 0;

// TLS переменная - у каждого потока своя копия
__thread int thread_counter = 0;
```

### Зачем это нужно:
```
Без TLS:
Поток 1: errno = 5 ──┐
Поток 2: errno = 11 ─┼──> Одна переменная errno
Поток 3: errno = 22 ─┘     Конфликты! Перезаписывают друг друга!

С TLS:
Поток 1: errno = 5  (свой экземпляр в TLS1)
Поток 2: errno = 11 (свой экземпляр в TLS2)
Поток 3: errno = 22 (свой экземпляр в TLS3)
         ↑ Нет конфликтов!
```

### В вашем trace.txt:

```
mmap(NULL, 12288, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS, -1, 0) = 0x7f8cdbb58000
```
**Выделение памяти для TLS** (12KB)

```
arch_prctl(ARCH_SET_FS, 0x7f8cdbb58740) = 0
```
**Установка регистра FS** - указывает на начало TLS текущего потока

### Структура TLS:
```
┌─────────────────────────────────┐
│  Thread Control Block (TCB)     │ ← FS регистр указывает сюда
├─────────────────────────────────┤
│  errno (thread-specific)        │
│  thread_id                      │
│  signal_mask                    │
│  robust_list (для futex)        │
├─────────────────────────────────┤
│  __thread переменные программы  │
│  Пользовательские TLS данные    │
└─────────────────────────────────┘
     ↑
  0x7f8cdbb58740
```

### Доступ к TLS переменным:
```asm
; На x86-64 доступ через FS регистр
mov rax, fs:[offset]  ; Чтение TLS переменной
```

### Применение:
1. **errno** - код ошибки последнего системного вызова
2. **Локаль** - настройки локализации для потока
3. **Random state** - состояние генератора случайных чисел
4. **Буферизация** - буферы для printf/scanf в каждом потоке
5. **OpenGL контексты** - у каждого потока свой контекст

### Пример использования:
````c
#include <stdio.h>
#include <pthread.h>

__thread int my_thread_var = 0;

void* thread_func(void* arg) {
    int id = *(int*)arg;
    my_thread_var = id * 100;
    printf("Thread %d: my_thread_var = %d\n", id, my_thread_var);
    return NULL;
}

int main() {
    pthread_t t1, t2;
    int id1 = 1, id2 = 2;
    
    pthread_create(&t1, NULL, thread_func, &id1);
    pthread_create(&t2, NULL, thread_func, &id2);
    
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    
    // Вывод:
    // Thread 1: my_thread_var = 100
    // Thread 2: my_thread_var = 200
    // У каждого потока своя копия!
    
    return 0;
}
````

### Связь TLS с системными вызовами в trace:
```
set_tid_address(0x7f8cdbb58d68) = 84495
```
Устанавливает адрес в TLS, где хранится Thread ID

```
set_robust_list(0x7f8cdbb58a20, 24) = 0
```
Устанавливает список robust futex в TLS для корректной обработки mutex при падении потока

---

**Итого**: 
- **ASLR** защищает от атак, рандомизируя адреса памяти
- **TLS** позволяет потокам иметь изолированные данные без конфликтов
